# 04 — Login + Auto-login Flow

Trace what happens when you click "Sign In".

## Files in this story

| File | Layer | Role |
|------|-------|------|
| `Views/Pages/Auth/LoginWindow.cs` | Window | Hosts the login frame, owns navigation between LoginPage ↔ ConfigPage |
| `Views/Pages/Auth/LoginPage.xaml` | XAML | The form layout |
| `Views/Pages/Auth/LoginPage.xaml.cs` | Code-behind | Resolves ViewModel, listens to events |

## Click-by-click flow

### 1. User types username

```xml
<TextBox Text="{Binding Username, Mode=TwoWay}" />
```

`Mode=TwoWay` means WinUI auto-syncs the TextBox to `LoginViewModel.Username`. Each keystroke updates the ViewModel.

### 2. User types password

Password is special — `PasswordBox` doesn't bind cleanly because of security. We use a code-behind handler:

```xml
<PasswordBox x:Name="PasswordInput" PasswordChanged="PasswordInput_PasswordChanged" />
```

```csharp
private void PasswordInput_PasswordChanged(object sender, RoutedEventArgs e)
{
    ViewModel.Password = PasswordInput.Password;
}
```

So password syncs to ViewModel via a manual event.

### 3. User clicks "Sign In"

```xml
<Button Command="{Binding LoginCommand}" Content="Sign In" />
```

`LoginCommand` is auto-generated by `[RelayCommand]` from this method:

```csharp
[RelayCommand(CanExecute = nameof(CanLogin))]
private async Task LoginAsync() { ... }
```

### 4. LoginViewModel.LoginAsync runs

```csharp
private async Task LoginAsync()
{
    if (IsBusy) return;
    ErrorMessage = null;

    if (string.IsNullOrWhiteSpace(Username) || string.IsNullOrWhiteSpace(Password))
    {
        ErrorMessage = "Please enter username and password.";
        return;
    }

    IsBusy = true;
    try
    {
        var isLoggedIn = await _authService.LoginAsync(Username.Trim(), Password);
        if (!isLoggedIn)
        {
            ErrorMessage = "Invalid username or password.";
            return;
        }
        SaveRememberedUsername();
        LoginSucceeded?.Invoke(this, EventArgs.Empty);
    }
    catch (Exception ex) { ErrorMessage = ex.Message; }
    finally { IsBusy = false; }
}
```

Notice: the ViewModel doesn't open a window itself. It just **fires `LoginSucceeded`**. The page handles the navigation.

### 5. AuthService.LoginAsync runs

```csharp
public async Task<bool> LoginAsync(string username, string password)
{
    var request = new LoginRequest { Username = username, Password = password };

    var result = await _graphQL.SafeExecuteAsync(() =>
        _graphQL.MutateAsync<LoginResponse>(AuthQueries.Login, request));

    if (!result.IsSuccess) return false;

    Token = result.Value!.Login.Token;
    CurrentUser = result.Value.Login.User;

    _graphQL.SetAuthToken(Token);
    SaveToken(Token);
    return true;
}
```

What happens:
1. Build a `LoginRequest` DTO
2. Call `MutateAsync` (HTTP POST) wrapped in `SafeExecuteAsync` (catches exceptions, returns Result<T>)
3. If success: extract token + user from the response
4. Set the token on the GraphQL client (will be sent as `Authorization: Bearer ...` on all future requests)
5. Save the token to LocalSettings (for next launch's auto-login)

### 6. GraphQL client sends HTTP POST

The query string from `AuthQueries.Login` is:
```graphql
mutation Login($username: String!, $password: String!) {
  login(username: $username, password: $password) {
    token
    user { userId username fullName role }
  }
}
```

Plus the variables `{ username: "admin", password: "admin123" }`.

`GraphQLClientService.SendAsync` POSTs this to the server URL with `Content-Type: application/json`.

### 7. Server responds

```json
{
  "data": {
    "login": {
      "token": "eyJhbGc...",
      "user": { "userId": "...", "username": "admin", "role": "Admin" }
    }
  }
}
```

The client deserializes this into `LoginResponse` (a DTO in `Services/Auth/Dto/`).

### 8. Page reacts to LoginSucceeded

```csharp
ViewModel.LoginSucceeded += OnLoginSucceeded;

private void OnLoginSucceeded(object? sender, EventArgs e)
{
    if (Application.Current is App app)
        app.OpenMainWindow();
}
```

`App.OpenMainWindow()` creates MainWindow, closes LoginWindow.

## Auto-login (no UI involved)

On next launch:

```csharp
// App.OnLaunched
var auth = Ioc.Default.GetRequiredService<IAuthService>();
var loggedIn = await auth.TryAutoLoginAsync();
if (loggedIn) { /* open MainWindow */ }
```

```csharp
// AuthService.TryAutoLoginAsync
public async Task<bool> TryAutoLoginAsync()
{
    var savedToken = LoadToken();
    if (string.IsNullOrEmpty(savedToken)) return false;

    _graphQL.SetAuthToken(savedToken);

    var result = await _graphQL.SafeExecuteAsync(() =>
        _graphQL.QueryAsync<MeResponse>(AuthQueries.Me));

    if (!result.IsSuccess || result.Value?.Me == null)
    {
        ClearToken();
        return false;
    }

    Token = savedToken;
    CurrentUser = result.Value.Me;
    return true;
}
```

The `me` query:
```graphql
query { me { userId username fullName role } }
```

If the saved token is still valid, the server returns the user info. We restore it locally and skip the login screen.

If the token is expired, the server returns `{ data: { me: null } }`, we wipe the saved token, and show the login screen.

## Logout

`MainWindow.xaml.cs → "Logout" case:`

```csharp
case "Logout":
    _authService.Logout();
    if (Application.Current is App app)
        app.OpenLoginWindow();
    Close();
    break;
```

`AuthService.Logout()`:
```csharp
public void Logout()
{
    CurrentUser = null;
    Token = null;
    _graphQL.SetAuthToken(null);
    ClearToken();
}
```

Wipes everything locally and opens LoginWindow.
